While Accuracy Alone is Misleading:
- Misleading Simplicity: single number holds critical datails
- Imbalanced Data: rare classes get ignored
- Unequal costs: different errors, different impacts


The Consusion Matrix
- True Positive: model correctly predcits positive class
- True Negative: model correctly predicts negative class
- False Positive: model incorrectly predicted positive class (Type 1 error)
- False Negative: model incorrectly predicted negative class (Type 2 error)


Accuracy (The Formula):
 Accuracy = (TP + TN) / (TP + TN + FP + FN)

This is the accuracy paradox-high accuracy masking complete failure on the minority class that actually matters.
Rule of Thumb: Only trust accuracy when your dataset has roughly equal class distributin and when false positives and false negatives carry similar business costs.


Precision: How Reliable Are Your Positive Predictions?

- The Question:
when my model says "positive", how often is it actually correct? Precision measures the proportion of true positives among all positives predictions.
- The Math:
precision focuses on minimizing alarms. It's calculated as TP / (TP + FP), measuring what percentage of predicted positives are truly positive.
- The Application:
high precision means fewer false positives. This is critical in spam detection, where marking legitimate emails as spam frustrates users, or in recommender systems where bad suggestions erode trusts.
Precision = TP / (TP + FP)


Recall: How Many Positives DId You Actually Catch?

While precision asks about the quality of positive predictions, recall measures completeness. Also called sensitivity or true positive rate, recall answers the question: "Of all the actual positive cases, what percentage did my model successfully identify?"

Formula:
Recall = TP / (TP + FN)

Recall focuses on minimizing misses by dividing true positives by all actual positives.
- Medical Diagnosis: missing a disease case cam be fatal-prioritize high recall
- Safety Systems: equipment failure prediction must catch all potential factors
- Search & Retrieval: information systems should return all requested documents


The Inevitable Precision-Recall Tradeoff:

Improving precision typically reduces recall, and vice versa. This isn't a flaw in your model-it's an inherent property of classification that stems from how we set decision thresholds.

The Threshold Effect:
Most classifiers output probability scores between 0 and 1. The decision threshold (typically 0.5) converts these probabilities into binary predictions. Lowering the threshold makes the model predict "positive" more often, increasing recall but decreasing precision. Raising it does the opposite.

- Lower threshold -> More positives predicted -> Higher Recall, lower precision
- Higher threshold -> Fewer positives predicted -> Higher precision, lower recall

Business Context Decides
The right baance depends entirely on the costs of different errors in your specific application:
- Favor Precision:
When false positives are expensive (spam filtering, drug side effects)
- Favor Recall:
When false negatives are dangerous (disease screening, fraud detection)
- Balance Both:
When both error typed matter equally (customer churn prediction)


F1-Score: Balancing Precision and Recall
It is the harmonic mean of precision and recall, which means it penalizes extreme imbalances more heavily than a simple average would.

F1 = 2 × ((Precision × Recall) / (Precision + Recall))

Unlike arithmetic mean, the harmonic mean gives more weight to lower values, so a model with highly unbalanced precision and recall receives a lower f1 score.


ROC Curve: Visualizing Performance Across All Thresholds

The Reciever Operating Characteristics curve provides a comprehensive view of classifier performance by plotting the True Positive Rate (recall) against the False Positive Rate at every possible classification threshold. This threshold-independent visualization reveals how well your model discriminates between classes.

Understanding the Plot
- X-axis: False Postive Rate = FP / (FP + TN)
- Y-axis: True Positive Rate = TP / (TP + FN)
- Diagonal line: Random guessing baseline
- Top-left corner: Perfect classifier (100% TPR, 0% FPR)

Practical Insights:
The ROC curve helps you visualize the precision-recall tradeoff we discussed earlier. By examining the curve, you can identify optimal thresholds for your specific use case:
- Choose your priority: decide acceptable FPR for your application
- Find the point: locate corresponding TPR on the curve
- Set threshold: use threshold value at that point
